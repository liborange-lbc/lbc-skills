## 验收报告模板（acceptance-report.md）

```markdown
# {Feature Name} 验收报告

> 日期: {date}
> 验收范围: 基于功能拆解清单，共 N 个功能子项

## 1. 测试验收矩阵

> 每个功能子项的测试覆盖情况和执行结果。

| FR 编号 | 功能描述 | 测试用例 | 测试类型 | 结果 | 验证说明 |
|---------|----------|----------|----------|------|----------|
| FR-A-001 | 创建订单 | test_FR_A_001_create_success | 集成 | ✅ PASS | 验证了订单创建、数据持久化、返回值 |
| FR-A-001 | 创建订单 | test_FR_A_001_empty_items | 边界 | ✅ PASS | 验证了空商品列表校验 |
| FR-A-002 | 幂等创建 | test_FR_A_002_duplicate | 集成 | ✅ PASS | 验证了重复 bizId 返回已有订单 |

## 2. 测试有效性审计

> 审计每个测试用例是否**真正在验证**其声称的系统功能。

| 测试用例 | 声称验证的功能 | 实际断言内容 | 有效性 | 理由 |
|----------|---------------|-------------|--------|------|
| test_FR_A_001_create_success | 订单创建 | 数据库记录存在 + 字段值正确 + HTTP 201 | ✅ 有效 | 断言覆盖了持久化和接口返回 |
| test_FR_A_002_duplicate | 幂等机制 | 返回相同订单 ID + 数据库无新增记录 | ✅ 有效 | 验证了幂等的核心行为 |
| test_FR_B_001_query | 订单查询 | result != null | ⚠️ 待改进 | 仅断言非空，未验证字段值 |

## 3. 覆盖度统计

| 指标 | 数值 | 说明 |
|------|------|------|
| 功能子项总数 | N | requirement-breakdown.md 中的总数 |
| 已覆盖子项 | N | 有至少一个测试用例 |
| 未覆盖子项 | N | 无测试用例 |
| 功能覆盖率 | xx% | 已覆盖/总数 |
| 测试总数 | N | 所有测试用例 |
| 测试通过数 | N | |
| 测试通过率 | xx% | |
| 测试有效数 | N | 有效性审计中判定为"有效" |
| 测试有效率 | xx% | |

## 4. 未覆盖子项

| FR 编号 | 功能描述 | 未覆盖原因 | 风险评估 |
|---------|----------|-----------|----------|
| FR-C-001 | {描述} | P2 延后实现 | 低风险 |

## 5. 验收结论

**结论**: ✅ 通过 / ⚠️ 有条件通过 / ❌ 不通过

**判定依据**:
- 功能覆盖率: xx%（≥90% 为通过）
- 测试通过率: xx%（100% 为通过）
- 测试有效率: xx%（≥80% 为通过）
- P0 子项覆盖率: xx%（必须 100%）

**遗留事项**:
{如有条件通过，列出遗留事项和跟进计划}
```
