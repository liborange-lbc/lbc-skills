# Knowledge Base Lookup Rules

Phase 2b 的知识库查阅规范。

## 查阅顺序

按以下优先级依次检索，不可跳过：

1. **已有系分文档** — 同模块或相关模块的历史系分
2. **API 契约** — 现有接口定义，不能破坏的契约
3. **数据字典** — 字段命名、类型约定、枚举值
4. **业务流程文档** — 确认对业务规则的理解

## 引用规则

- 所有引用必须标注来源：`[来源: 文档名, 章节/页码]`
- 引用代码时标注：`[来源: 文件路径:行号]`
- 不确定的信息标注为 `[待确认]`

## 冲突处理

| 冲突类型 | 处理方式 |
|----------|----------|
| 文档 vs 代码 | **代码为准**，标记差异并建议更新文档 |
| 旧系分 vs 新需求 | 以新需求为准，说明与旧设计的差异点 |
| 不同文档互相矛盾 | 列出两种说法，请用户裁决 |

## 查阅不到时

- 明确写出"未找到相关文档，以代码为准"
- 不要编造或猜测未经验证的信息
- 如果是关键信息缺失，暂停并通过 AskUserQuestion 向用户确认
