# 编码实现指南

> coder 子Agent 在 Phase 5 使用。

## 输入

- `需求清单.md` — 作为实现 checklist
- `系分文档.md` — 实现依据
- `需求追踪矩阵.md` — 需求→接口→数据的映射
- `经验库.md` — 避免重犯已知问题（如存在）
- EXTENSION_DOCS.coding — 代码军规等扩展文档

## 批量实现流程

按 BATCH_SIZE 分批：

```
for each batch:
  1. 读取本批需求条目 + 对应系分章节
  2. 实现代码
  3. 为每条需求编写对应测试
  4. 更新需求清单中对应条目状态
  5. 写入 log/phase5-编码.md
```

## 行为约束

1. **严格按系分实现** — 不得擅自修改设计方案。有疑问写入日志，由主Agent转达
2. **每条需求 ≥ 1个测试** — 测试覆盖需求的验收标准
3. **增量追加** — 不重写已完成的代码
4. **先读经验库** — 开始编码前先读 `经验库.md`，避免重犯
5. **匹配现有风格** — 命名、缩进、模式与项目现有代码一致
6. **扩展文档遵循** — 如有代码军规，每条规则作为编码约束

## 修正模式（Resume）

当主Agent Resume 本Agent修复CR问题时：

1. 读取 CR 报告中的具体修复指令
2. **定点修复** — 只改评审指出的问题，不顺手改其他地方
3. 修复完成后写入 `log/phase6-迭代-{N}.md`
4. 更新经验库 — 如果修复揭示了可复用的模式

- ...
