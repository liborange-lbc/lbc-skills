# 设计文档模板

> designer 子Agent 在 Phase 3 使用。先出骨架让用户确认，再填充。

## 文档骨架

```markdown
# [Feature Name] 系统设计

## 1. 背景与目标
### 1.1 业务背景
### 1.2 设计目标
### 1.3 术语表

## 2. 需求追踪矩阵
| 需求ID | 描述 | 设计章节 | 接口 | 数据变更 | 状态 |
|--------|------|---------|------|---------|------|

## 3. 整体方案
### 3.1 架构图
### 3.2 模块划分
### 3.3 核心流程（时序图）

## 4. 详细设计
### 4.1 接口设计
### 4.2 数据模型变更
### 4.3 缓存设计（如需要）
### 4.4 消息设计（如需要）

## 5. 非功能设计
### 5.1 性能方案
### 5.2 高可用方案
### 5.3 安全方案

## 6. 兼容性与发布
### 6.1 向前/向后兼容
### 6.2 数据迁移方案
### 6.3 灰度/回滚方案

## 7. 风险与应对

## 8. 附录
### 8.1 代码阅读报告引用
### 8.2 参考文档
```

## 接口设计格式

```markdown
#### [接口名称]

- **Path**: POST /api/v1/xxx
- **Auth**: 鉴权方式
- **Idempotent**: 幂等机制
- **关联需求**: R01, R02
- **Request**: 请求体定义
- **Response**: 响应体定义
- **Error Codes**: 错误码
- **Rate Limit**: 限流策略
```

## 数据模型变更格式

```markdown
#### 表名 / Collection / Schema

**变更类型**: 新建 / 加字段 / 加索引
**关联需求**: R01

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|

**索引变更**: ...
**数据迁移**: 是否需要？耗时？锁表？
**容量预估**: 日增量、总量
```
