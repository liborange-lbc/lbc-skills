# 经验库规范

> `经验库.md` 位于 `{OUTPUT_DIR}/` 根目录，全局共享，跨需求累积。

## 初始化

- **时机**: 首个需求的 Phase 2 完成后创建（空文件 + 框架）
- **创建者**: 主Agent
- **初始内容**:

```markdown
# 经验库

> 修正循环中沉淀的可复用模式。跨需求共享。

## 架构类

## 编码类

## 安全类

## 性能类

## 测试类
```

## 追加规则

### 何时追加
- 每轮**修正循环完成后**（Phase 4/6 的修正），将本轮修复揭示的模式追加
- 主Agent 负责指示相关子Agent（designer/coder）追加

### 追加内容的抽象层级

| 层级 | 示例 | 是否合适 |
|------|------|---------|
| 太具体 | "page03 第42行 left 应该是 120" | ❌ |
| 太模糊 | "代码要写好" | ❌ |
| **合适** | "分页查询必须处理空结果，否则上层 map 报 NPE" | ✅ |
| **合适** | "该项目外部支付接口超时 3s，调用处必须有降级" | ✅ |

### 格式

每条经验一行，归入对应类别：

```markdown
## 编码类
- 分页查询必须处理空结果，否则上层 map 报 NPE
- 该项目用 early return 风格，不要用 if-else 嵌套
```

## 使用规则

| 角色 | 使用方式 |
|------|---------|
| coder | Phase 5 编码前**必读**，避免重犯已知问题 |
| designer | Phase 3 设计时参考，了解已知的技术约束 |
| 其他子Agent | 按需读取 |

## 维护

- **不删除** — 经验只增不减（除非项目技术栈彻底变更）
- **去重** — 追加前检查是否已有类似条目
- **不记行号** — 只记可复用的模式
