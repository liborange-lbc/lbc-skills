# Phase 1 需求澄清 — 流水线可视化Dashboard

## 基本信息

- 面试员: interviewer (AI)
- 面试时间: 2026-05-01
- 面试轮数: 2 轮
- 总问题数: 8 题
- 状态: 已完成

## 第一轮面试记录

### Q1: 业务背景 — 展示设计理念 vs 运行时状态？

**用户回答**: 两者兼顾 — 默认展示设计理念，运行时能读取实际数据展示实时状态

**需求影响**: Dashboard 需要两种模式：静态设计理念展示 + 运行时实时数据展示

### Q2: 展示粒度 — 阶段概览 vs 运行时详情？

**用户回答**: 运行时详情 — 展示每个 Agent 的实际输入摘要、执行进度、输出文件内容摘要（解析 log 目录文件）

**需求影响**: 需要 Agent 级别的 IO 展示，需解析 log 目录文件提取数据

### Q3: 数据源 — 静态配置 vs 扫描 log 目录？

**用户回答**: 扫描 log 目录 — 定时扫描 `.ai-coding/{需求名称}/log/` 目录下的文件（需要轻量后端）

**需求影响**: 需要 Python 后端服务扫描文件系统，前端定时轮询

### Q4: 用户角色 — 开发者本人 vs 团队共享？

**用户回答**: 开发者本人 — 在本地浏览器中查看

**需求影响**: 无需多用户支持、权限控制，本地运行即可

## 第二轮面试记录

### Q5: 数据流架构 — 纯前端读取 vs 后端服务？

**用户回答**: Python HTTP Server — 用 Python 脚本扫描 log 目录返回 JSON，配合 http.server 提供服务

**需求影响**: 后端技术选型确定为 Python 标准库

### Q6: 视觉风格？

**用户回答**: 简洁白色 — 浅色背景、清晰排版、类似 GitHub/Notion 风格

**需求影响**: UI 设计约束确定

### Q7: 技术约束 — 可否引入依赖？

**用户回答**: 零依赖 — 不引入任何 npm 包或 CDN，纯原生 HTML/CSS/JS

**需求影响**: 前端零依赖约束，Markdown 渲染需手写解析器

### Q8: 验收标准？

**用户回答** (全选 + 补充):
1. 能看到 8 Phase 流程图
2. 能看到每个 Agent 的 IO
3. 实时刷新有效（每 5 秒）
4. 设计理念可视化
5. **设计理念简单化，融合到实际运行即能理解**（不要单独的设计理念板块）
6. **多 agent 并行执行时可以看到所有 agent 信息**
7. **添加文件引用，产出文档可链接方式点击查看详情（支持 Markdown 解析）**

**需求影响**: 验收标准 5/6/7 为用户补充的额外需求，需单独拆条

## 关键决策摘要

| 决策项 | 决策结果 | 理由 |
|--------|---------|------|
| 前端技术 | 纯原生 HTML/CSS/JS | 零依赖约束 |
| 后端技术 | Python http.server | 轻量、标准库即可 |
| 数据源 | 扫描 log 目录文件 | 与流水线现有日志机制对接 |
| 视觉风格 | 简洁白色/GitHub 风格 | 开发者偏好 |
| 刷新策略 | 前端 5 秒轮询 | 简单可靠 |
| 设计理念展示 | 融入流程节点 | 用户明确不要独立板块 |
| Markdown 渲染 | 手写 JS 解析器 | 零依赖约束 |
