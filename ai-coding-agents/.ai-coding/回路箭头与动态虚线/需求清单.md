# 需求清单 — 回路箭头与动态虚线

## 需求条目

| ID | 需求描述 | 验收标准 | 状态 |
|----|---------|---------|------|
| R01 | 回路箭头触发：P4/P6 在 fail 或 iterating 状态时展示回路箭头（P4→P3, P6→P5） | SVG 中可见回路曲线路径，仅在对应状态触发 | ✅ |
| R02 | 回路箭头标记循环次数：箭头旁显示 "x{N}" 格式的迭代计数，替换当前 "Resume" 文字 | 回路箭头旁显示如 "x2"，数据来源于 iteration_count 字段 | ✅ |
| R03 | 正向箭头虚线化：所有 P1→P2→...→P8 的正向连线改为虚线样式 | 所有 edge-path 使用 stroke-dasharray 虚线 | ✅ |
| R04 | 正向箭头流动动画：正向虚线箭头添加 stroke-dashoffset 动画，流动方向为左→右 | CSS 动画实现虚线流动效果 | ✅ |
| R05 | 回路箭头流动动画：回路虚线箭头添加 stroke-dashoffset 动画，流动方向为回退方向 | 回路箭头动画方向与正向相反 | ✅ |
| R06 | 动画速度分级：活跃状态（running/iterating 阶段相关箭头）快速流动，非活跃状态慢速流动 | running/iterating 相关箭头动画约 1-2s 周期，其余约 3-4s 周期 | ✅ |
| R07 | 回路箭头颜色区分：回路箭头使用 #FF9800 橙色，与正向箭头视觉区分 | 回路箭头 stroke 和文字 fill 为橙色 | ✅ |

## 技术约束

- 纯前端修改：仅涉及 app.js 和 style.css
- 后端数据已具备：server.py 已返回 iteration_count、max_iteration、iterations 字段
- 兼容设计理念模式：无数据时不展示回路箭头（全 pending 无触发条件）
- 保持现有 SVG 坐标系和节点布局不变
