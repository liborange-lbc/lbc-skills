# 测试验证指南

> tester 子Agent 在 Phase 7 使用。

## 输入

- 代码文件 + 测试文件
- `需求清单.md` — 逐条确认测试通过
- EXTENSION_DOCS.testing — 测试规范等
- GitNexus 索引 — 影响面分析

## 执行流程

### 1. 运行测试套件

```bash
{build_tool} test    # 运行全部测试
```

### 2. 检查覆盖率

目标: ≥ 80%。低于目标时列出未覆盖的关键路径。

### 3. 静态检查

```bash
# lint + typecheck（根据项目技术栈）
```

### 4. GitNexus 影响面分析

**关键步骤**: 用 GitNexus 检查本次变更的影响范围，确认测试覆盖所有受影响模块。

```bash
# 检测本次变更影响了哪些符号和执行流
gitnexus detect-changes --scope all --base-ref {base-branch} -r {project-name}

# 对关键变更符号做上游影响分析
gitnexus impact {ChangedSymbol} --direction upstream --depth 3 --include-tests -r {project-name}
```

**核对规则**:
- detect-changes 列出的每个受影响执行流，必须有对应测试覆盖
- impact 分析中的上游依赖方，如果没有测试覆盖，标记为 **HIGH** 风险
- 将影响面分析结果写入测试报告的"影响面覆盖"章节

**降级方案**: 如果 GitNexus 不可用（未安装、非 git 仓库等），跳过影响面分析，在测试报告中标注 "GitNexus N/A"，通过 `git diff --stat` 手动识别变更文件范围

### 5. 需求级验证（逐条、逐Case）

**每条需求至少一个 test case，复杂需求需多个 case 覆盖正常/异常/边界。**

#### Test Case 规范

每个 case 必须包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| Case ID | `{需求ID}-TC{序号}` | R01-TC01 |
| 测试目标 | 验证什么行为 | 正常路径：8个Phase节点渲染 |
| 代码位置 | `文件名:类/函数名:起始行-结束行` | app.js:renderPipeline:120-145 |
| 测试入参 | 调用时传入的参数/前置状态 | `phases=[{id:"P1",status:"pass"},...8项]` |
| 预期出参 | 期望的返回值/DOM 变化/副作用 | SVG 中存在 8 个 `<g class="phase-node">` 元素 |
| 实际出参 | 真实执行结果 | SVG 中存在 8 个节点，class 和颜色正确 |
| 验证方式 | 如何确认结果 | 代码检查 / 运行时 DOM 查询 / curl 验证 / 断言 |
| 结论 | PASS / FAIL | PASS |

#### 验证方式分类

| 验证类型 | 适用场景 | 具体操作 |
|----------|---------|---------|
| **代码静态检查** | 逻辑存在性验证 | Grep/Read 代码确认函数/分支存在 |
| **运行时请求验证** | API 接口行为 | `curl` 请求 + 断言响应 JSON 字段 |
| **DOM/UI 验证** | 前端渲染结果 | 浏览器 DevTools 或 JS 注入检查 |
| **语法检查** | 代码可执行性 | `python3 -c "py_compile.compile()"` / `node --check` |
| **安全测试** | 路径穿越/XSS | 构造恶意入参验证拒绝 |

#### 每条需求的 Case 数量指引

| 需求类型 | 最少 Case 数 | 覆盖维度 |
|----------|-------------|---------|
| 渲染/展示类 | 2 | 正常渲染 + 空数据/异常状态 |
| 接口/API类 | 3 | 正常响应 + 参数缺失 + 资源不存在 |
| 交互/事件类 | 2 | 正常触发 + 边界条件 |
| 安全类（注入/XSS/越权） | 2 | 正常请求 + 恶意输入 |
| 非功能需求 | 1 | 约束验证 |

## 测试不通过处理

- 测试失败 → 主Agent Resume coder 修复 → 重新运行测试
- 影响面未覆盖 → 主Agent Resume coder 补充测试
- 最多 MAX_ITERATION 轮
