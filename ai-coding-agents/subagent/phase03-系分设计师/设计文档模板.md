# 设计文档模板

> designer 子Agent 在 Phase 3 使用。先出骨架让用户确认，再填充。

## 文档骨架

```markdown
# [功能/模块名称] 系分设计

## 1. 需求与范围
- 背景与目标
- 核心功能
- 约束与非功能要求
- 排除范围

## 2. 架构与模块
### 功能架构
（Mermaid graph 图 — 模块划分、层次关系）
- 各模块/层级职责说明

### 应用集成架构
（Mermaid flowchart 图 — 应用间调用关系、协议标注）
- 集成关系说明

### 部署架构
（Mermaid graph 图 — 机房、集群、负载均衡、数据库主从、WAF 等）
- 部署策略说明（容器化/虚拟机、同城双机房、高可用方案）

## 3. 数据模型与存储
### 数据模型ER图
（Mermaid erDiagram — 核心实体、关系）
### 表结构定义
#### 表名
| 属性编码 | 属性名称 | 数据类型 | 约束 | 注释 |
|------|------|------|------|------|
（每个字段一行，不合并）

## 4. 接口设计
#### oneapi
##### 接口名称
- URI: /webapi/xxx
- method: POST/GET
- 接口描述: 
- 入参

| 参数名称 | 类型 | 是否必填 | 描述 |
|------|------|------|------|

- 出参

| 参数名称 | 类型 | 描述 |
|------|------|------|

#### 对外 OpenAPI
（同上格式）

#### 内部接口
（方法签名 + 详细出入参）

#### 依赖服务接口
（外部系统接口定义）

## 5. 功能模块设计
### 5.N: 模块名称
#### 5.N.M 功能点名称
- 处理时序图
（Mermaid sequenceDiagram — 调用流程）
- 调用说明
- 关键设计（核心算法、类图、状态图等）

## 6. 非功能性需求设计
### 6.1 高可用性
### 6.2 可扩展性
### 6.3 稳定性/可靠性
### 6.4 安全性设计
#### 6.4.1 账户系统方案
#### 6.4.2 授权&访问控制
##### 6.4.2.1 水平权限检查
##### 6.4.2.2 垂直权限检查
##### 6.4.2.3 登录态检查
#### 6.4.3 数据防护方案
##### 6.4.3.1 敏感数据加密存储
##### 6.4.3.2 敏感数据展示脱敏
### 6.5 监控/统计/日志/告警

## 7. 变更三板斧
### 7.1 可监控
### 7.2 可灰度
### 7.3 可应急
```

## 接口设计格式

```markdown
#### [接口名称]

- **URI**: POST /api/v1/xxx
- **method**: POST/GET/PUT/DELETE
- **接口描述**: 功能说明
- **Auth**: 鉴权方式
- **Idempotent**: 幂等机制
- **关联需求**: R01, R02
- **入参**:

| 参数名称 | 类型 | 是否必填 | 描述 |
|------|------|------|------|

- **出参**:

| 参数名称 | 类型 | 描述 |
|------|------|------|

- **Error Codes**: 错误码定义
- **Rate Limit**: 限流策略（如有）
```

## 数据模型变更格式

```markdown
#### 表名

**变更类型**: 新建 / 加字段 / 加索引
**关联需求**: R01

| 属性编码 | 属性名称 | 数据类型 | 约束 | 注释 |
|------|------|------|------|------|

**索引变更**: ...
**数据迁移**: 是否需要？耗时？锁表？
**容量预估**: 日增量、总量
```

## 时序图

使用 Mermaid sequenceDiagram，每条核心流程一张图。

## 状态图

使用 Mermaid stateDiagram-v2，有状态转换的实体必须画状态图。
