# 知识查阅规则

## 查阅优先级

1. 已有系分文档（同/相关模块历史）
2. API契约（现有接口定义）
3. 数据字典（命名、类型、枚举）
4. 业务流程文档（业务规则）
5. EXTENSION_DOCS 中声明的扩展文档

## 三条硬性规则

1. **标注来源** — 所有引用: `[来源: 文档名, 章节]` 或 `[来源: 文件:行号]`
2. **代码为准** — 文档与代码冲突时以代码为准，标记差异
3. **不得编造** — 查不到写"未找到相关文档，以代码为准"

## 冲突处理

| 冲突类型 | 处理方式 |
|----------|----------|
| 文档 vs 代码 | **代码为准**，标记差异 |
| 旧系分 vs 新需求 | 新需求为准，说明变化点 |
| 不同文档矛盾 | 列出两种说法，请用户裁决 |
| 扩展文档 vs 代码 | 代码为准，但规范性要求（军规）优先 |
