# 学习系统健康检查

## 触发条件

用户说："learning status" / "学习状态"

## 执行

1. 读取 `~/.claude/learning/.index.json`
2. 统计各目录文件数：
   - `sessions/` → L1 数量
   - `patterns/` → L2 数量（排除 `.archive/`）
   - `~/.claude/rules/common/learned/` → L3 规则数量
   - `~/.claude/skills/learned/` → L3 技能数量
3. 读取模式文件获取置信度分布
4. 检查最后捕获日期

## 输出格式

```
知识编译器状态
━━━━━━━━��━━━━

L1 会话：  {N} 个已捕获（最近：{日期}）    {状态图标}
L2 模式：  {N} 个活跃（{low}/{med}/{high} 置信度）  {状态图标}
L3 规则：  {N} 个已晋升（位于 ~/.claude/rules/common/learned/）
L3 技能：  {N} 个已生成（位于 ~/.claude/skills/learned/）

管道健康度：
  L1→L2 转化率：{百分比}%（{模式数}/{会话数}）   {状态图标}
  L2→L3 晋升率：{百分比}%（{规则+技能数}/{模式数}） {状态图标}

{状态图标} 含义：
  正常：健康（持续捕获、模式在成熟）
  注意：需要关注（7天以上未捕获，或模式停滞）
  警告：层级为空或管道断裂

建议操作：
- {操作1，如"运行提炼——检测到3个会话聚类"}
- {操作2，如"运行复盘——2个模式已达 high 置信度，可以晋升"}
- {操作3，如"10天未捕获会话——考虑手动捕获"}
```
