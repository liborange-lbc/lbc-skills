# 自动捕获工作流

会话结束时静默运行，提取 L1 会话摘要（如果会话包含可学习内容）。

> 注意：此工作流已被 `auto-pipeline.md` 取代，仅作为第一阶段的独立参考保留。

## 触发条件

对话即将结束（用户说再见、/exit、或会话空闲）。

## 执行步骤

### 步骤 1：评估会话价值

扫描对话中的信号：
- [ ] ≥8 条用户消息（实质性工作）
- [ ] 调试了 bug 或解决了错误
- [ ] 尝试了新技术、工具或方法
- [ ] 用户给出了纠正或反馈
- [ ] 完成了多文件实现
- [ ] 做出了架构或设计决策
- [ ] 进行了性能优化

**如果没有任何信号 → 跳过。不捕获琐碎会话。**

### 步骤 2：提取会话摘要

创建 L1 文件，路径 `~/.claude/learning/sessions/YYYY-MM-DD-{slug}.md`：

```markdown
---
date: YYYY-MM-DD
duration: ~Xm
domain: {项目/工具/语言领域}
signals: [步骤1中匹配到的信号]
---

# 会话：{描述性标题}

## 完成的工作
- {1-3 条具体要点}

## 有效的方法
- {成功的方法，附带上下文}

## 失败或被纠正的部分
- {失败的尝试、用户纠正、死胡同}

## 核心洞察
{一句话：本次会话学到的非显而易见的东西。如果没有非显而易见的内容，写"常规工作，无新洞察。"}

## 涉及的文件
- {相关文件路径}
```

### 步骤 3���检查模式候选

扫描 `~/.claude/learning/sessions/` 中已有的 L1 会话：
- 同一 `domain` 出现 ≥2 ��
- 相似的 `核心洞察` 主题
- 重复的 `失败或被纠正的部分` 模式
- 反复出现的 `有效的方法`

**如果发现模式候选：**
1. 在会话文件中标注 `## 模式信号：{简要描述}`
2. 不自动创建 L2——仅标记，留给下一次 `distill` 或 `review` 工作流处理

### 步骤 4：更新索引

更新 `~/.claude/learning/.index.json`。

### 步骤 5：简短通知

输出一行：
```
[学习] 已捕获：{会话标题} | {模式信号（如有），否则"无新模式"}
```

如果会话被跳过，不输出任何内容。
