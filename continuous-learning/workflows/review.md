# 模式复盘工作流

定期复盘 L2 模式：验证、更新置信度、晋升或退役。

## 触发条件

用户说："review patterns" / "复盘" / "review learning"

## 步骤 1：加载当前状态

1. 读取 `~/.claude/learning/.index.json` 获取计数
2. 读取 `~/.claude/learning/patterns/` 中的所有 L2 模式文件
3. 读取最近 30 天的 L1 会话 `~/.claude/learning/sessions/`

## 步骤 2：展示复盘看板

```
学习系统复盘
━━━━━━━━━━━━
会话（L1）：{N} 个，最近30天 {N} 个
模式（L2）：{N} 个活跃，{N} 个候选
规则（L3）：{N} 个已晋升

需要关注的模式：
┌──────────────────────────────────────────────────┐
│ # │ 模式名称             │ 置信度   │ 证据数 │ 存在时间 │
│ 1 │ {标题}               │ low      │ 2      │ 14天     │
│ 2 │ {标题}               │ medium   │ 4      │ 30天     │
│ 3 │ {标题}               │ high     │ 5      │ 7天      │ ← 可以晋升
└──────────────────────────────────────────────────┘

操作：[晋升 N] [补充证据 N] [退役 N] [查看详情 N]
```

## 步��� 3：逐模式复盘

用户选择的每个模式：

### 如果存在新证据（近期会话匹配该领域）
- 展示尚未关联的匹配会话
- 询问："这个会话是否支持该模式？"
- 如果是 → 添加到证据表，提升置信度
- 如果否 → 记录为反例

### 如果 confidence=high 且 status=active
- 建议晋升为 L3（规则或技能）
- 起草规则/技能内容
- 请用户确认或编辑

### 如果模式已过时（60+ 天无新证据）
- 询问："仍然有效？[保留 / 退役 / 与其他模式合并]"
- 退役的模式移动到 `~/.claude/learning/patterns/.archive/`

## 步骤 4：跨模式分析

寻找：
- **可合并的模式**：两个模式描述同一件事但用词不同
- **矛盾的模式**：模式之间存在冲突——标记给用户解决
- **缺失的模式**：某些领域有很多 L1 会话但没有 L2 模式

报告发现：
```
跨模式洞察：
- {模式A} 和 {模式B} 可能是同一个模式（重叠：{详情}）
- 领域"{X}"有 {N} 个会话但无模式——建议提炼
```

## 步骤 5：总结

```
复盘完成：
- 复盘模式数：{N}
- 补充证据：{N} 条关联
- 置信度升级：{列表}
- 晋升为 L3：{N}
- 退役：{N}
- 建议下次复盘时间：{日期，14天后}
```
